# AI World Builder - 游戏设计规格书

## 1. 项目概述

**项目名称**: AI World Builder  
**类型**: 实时AI管理模拟游戏  
**核心玩法**: 玩家作为世界的管理者，通过自然语言指令让AI创建和管理一个持续发展的虚拟世界  
**技术路线**: Python后端 + HTML/JS前端，LLM驱动世界逻辑

## 3. 核心模块设计

### 3.1 LLM 接入层 (LLM Adapter)

**支持模式**:
- 本地模型: Ollama API (默认 `qwen2.5:7b` 或 `llama3.2:3b`)
- 云端API: OpenAI / Anthropic / Ollama

**配置方式**:
```json
{
  "provider": "ollama",  // 或 "openai", "anthropic"
  "model": "qwen2.5:7b",
  "base_url": "http://localhost:11434",
  "api_key": "optional"
}
```

### 3.2 世界状态管理 (World Manager)

**核心数据结构**:
```python
class Entity:  # 世界中的实体
    id: str
    type: str          # "land", "creature", "plant", "building", "resource"
    x: int
    y: int
    properties: dict   # 颜色、能量、状态等
    name: str
    description: str

class World:
    entities: dict[id -> Entity]
    tick: int           # 世界时间
    history: list       # 事件日志
    rules: dict         # 运行规则（AI管理）
```

**世界规则**:
- 由AI根据玩家指令动态创建和维护
- 存储为自然语言描述的规则集
- 每个tick执行规则检查和状态更新

### 3.3 Agent 引擎 (Agent Engine)

**指令解析流程**:
1. 接收玩家指令
2. 加载世界当前状态（实体列表、规则、历史）
3. 构造 Prompt 包含:
   - 系统提示（角色定义）
   - 世界状态摘要
   - 可用工具列表
   - 玩家指令
4. 调用 LLM
5. 解析 LLM 响应为动作
6. 执行动作，更新世界状态
7. 返回结果给前端

**可用工具** (受限):
- `create_entity`: 创建新实体
- `update_entity`: 修改实体属性
- `delete_entity`: 删除实体
- `get_world_state`: 获取世界状态
- `add_rule`: 添加世界规则
- `search_web`: 搜索网页（可选）
- `log_event`: 记录事件

### 3.4 自动推进机制

- 每个 `tick` (可配置，默认5秒) 执行一次
- AI 分析当前世界状态和规则
- 决定需要执行的自动动作
- 更新实体状态（移动、生长、变化）
- 生成事件描述

### 4.2 实体渲染规则

| 类型 | 颜色 | 文字 | 备注 |
|------|------|------|------|
| 陆地 | #8B4513 (棕色) | 无 | 基础地形 |
| 植物 | #228B22 (绿色) | 植 | 可生长 |
| 生物 | #FF6347 (红色) | 生 | 可移动 |
| 建筑 | #4682B4 (蓝色) | 建 | 静态 |
| 资源 | #FFD700 (金色) | 矿 | 可采集 |

### 4.3 通信协议

**WebSocket 消息格式**:
```json
// 客户端 -> 服务器
{"type": "command", "content": "创建一片森林"}
{"type": "ping"}

 // 服务器 -> 客户端
{"type": "state", "data": {...}}
{"type": "event", "message": "..."}
{"type": "error", "message": "..."}
```

## 5. 启动流程

1. 玩家输入一句话启动世界（如"创造一个花园"）
2. AI 解析指令，创建初始实体
3. 世界开始自动推进
4. 玩家持续发出指令扩展世界
5. 世界无限扩张

---

## 6. 技术栈

- **后端**: Python 3.10+ / FastAPI / WebSocket / Uvicorn
- **LLM**: Ollama (本地) 或 OpenAI API
- **前端**: 原生 HTML5 + Canvas + WebSocket
- **存储**: JSON 文件（世界状态、记忆）

## 8. 安全限制

- AI 仅能操作虚拟世界状态
- 无文件系统访问权限
- 无执行系统命令能力
- 外部网络访问可选且受限
- 所有操作通过工具白名单控制
